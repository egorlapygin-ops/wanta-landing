# Пошаговая инструкция: выложить сайт на Vercel и подключить домен

## Часть 1: Зарегистрироваться на Vercel и задеплоить сайт

### Шаг 1. Регистрация на Vercel

1. Откройте в браузере: **https://vercel.com**
2. Нажмите **Sign Up** (Зарегистрироваться).
3. Выберите **Continue with GitHub** — войдите через ваш аккаунт **egorlapygin-ops** (тот, где лежит репозиторий).
4. Разрешите Vercel доступ к GitHub, если попросит.

---

### Шаг 2. Создать новый проект из репозитория

1. На главной Vercel нажмите **Add New…** → **Project** (или **Import Project**).
2. В списке репозиториев найдите **wanta-landing** (репозиторий из egorlapygin-ops).
3. Если репозитория нет — нажмите **Adjust GitHub App Permissions** и выдайте Vercel доступ к нужным репозиториям (или ко всем).
4. Нажмите **Import** рядом с **wanta-landing**.

---

### Шаг 3. Настройки проекта (обычно менять не нужно)

1. **Project Name** — можно оставить `wanta-landing` или изменить.
2. **Framework Preset** — должно быть **Next.js** (Vercel подставит сам).
3. **Root Directory** — оставьте пустым.
4. **Build and Output Settings** — не трогайте.

Нажмите **Deploy**.

---

### Шаг 4. Дождаться сборки

1. Vercel начнёт собирать проект (1–3 минуты).
2. Когда сборка закончится, появится зелёная галочка и ссылка вида:
   **https://wanta-landing-xxxx.vercel.app**
3. Нажмите **Visit** — откроется ваш лендинг.

Сайт уже в сети по этой ссылке. Дальше — по желанию подключить свой домен.

---

## Часть 2: Подключить свой домен (по желанию)

### Шаг 5. Открыть настройки домена в проекте

1. В Vercel откройте ваш проект **wanta-landing**.
2. Перейдите на вкладку **Settings** (Настройки).
3. В левом меню выберите **Domains**.

---

### Шаг 6. Добавить домен

1. В поле введите ваш домен, например: **wanta.ru** или **landing.wanta.ru**.
2. Нажмите **Add**.
3. Vercel покажет, какие записи нужно добавить у регистратора домена.

---

### Шаг 7. Настроить DNS у регистратора

1. Зайдите в панель, где вы покупали домен (Reg.ru, Timeweb, Cloudflare, Namecheap и т.д.).
2. Найдите раздел **DNS**, **Управление зоной**, **DNS-записи**.
3. Добавьте запись так, как просит Vercel:
   - Обычно для корня домена (**wanta.ru**) — запись **A** с именем **@** и значением **76.76.21.21**.
   - Для поддомена (**www** или **landing**) — запись **CNAME** с именем **www** (или **landing**) и значением **cname.vercel-dns.com** (точное значение покажет Vercel).
4. Сохраните изменения.

---

### Шаг 8. Дождаться обновления DNS

1. Обновление DNS занимает от 5 минут до 24–48 часов (чаще всего 10–30 минут).
2. В Vercel в разделе **Domains** напротив домена появится статус **Valid** (и галочка), когда всё заработает.
3. Откройте ваш домен в браузере — должен открываться лендинг.

---

## Краткая шпаргалка

| Что сделать | Где |
|-------------|-----|
| Зарегистрироваться | vercel.com → Sign Up with GitHub |
| Импортировать репозиторий | Add New → Project → wanta-landing → Import |
| Задеплоить | Deploy (настройки по умолчанию) |
| Открыть сайт | Visit по ссылке типа xxx.vercel.app |
| Свой домен | Settings → Domains → добавить домен → настроить DNS у регистратора |

---

## Если что-то пошло не так

- **Сборка упала с ошибкой** — откройте в Vercel вкладку **Deployments**, нажмите на упавший деплой и посмотрите **Build Logs**. Часто помогает проверить, что в репозитории есть `package.json` и команда `npm run build` выполняется локально.
- **Домен не привязывается** — убедитесь, что DNS-записи введены точно так, как в Vercel, и подождите до 24 часов.
- **Нужна помощь по конкретному регистратору** — напишите, какой у вас (Reg.ru, Timeweb и т.д.), можно расписать шаги под него.
